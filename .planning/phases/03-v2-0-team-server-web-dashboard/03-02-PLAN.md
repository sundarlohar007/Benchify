    **Step A — Port analytics engine Dart→Rust (D-18):**
    Write `server/src/services/mod.rs`: `pub mod analytics;`
    Write `server/src/services/analytics.rs`: Port `computeSessionStats()` from `analytics_service.dart` line-by-line to Rust. Exact formulas to replicate:

    ```rust
    pub fn compute_session_stats(samples: &[MetricSample], session_id: Uuid, duration_ms: i64, markers: &[Marker]) -> SessionStats
    ```

    Key computations (must match Dart output exactly):
    1. **FPS**: Filter `s.fps` non-null values, compute via `FpsAnalytics::compute()`: median, min, max, 1% low, stability %, p95 frame time, histogram JSON, variability index
    2. **CPU**: Mean and peak of `cpu_app_pct`, `cpu_app_pct_freq_norm`
    3. **Memory**: Mean and peak of `memory_pss_kb`, plus all 8 subsection means/peaks (java, native, graphics, stack, code, system, webview), memory growth (last - first), memory trend (linear regression slope in KB/min)
    4. **GPU**: Mean and peak of `gpu_pct`
    5. **Battery/Power**: Filter non-charging samples. Compute drain pct (first - last), drain per hour, temp max, mAh consumed (trapezoidal integration: `(mA1 + mA2) / 2 * dt / 3600`), avg mW, total mWh, estimated playtime (4000 mAh / avg mA). Set `has_charging_period = 1` if any sample has `charging == 1`.
    6. **Jank**: Sum jank_count, jank_small_count, jank_big_count, jank_ratio_count across all samples. Jank per minute = total_jank / duration_minutes.
    7. **Network**: Delta TX/RX (last - first) / 1024 for each interface (total, wifi, cellular, other). Avg Kbps = total KB / duration_seconds for wifi and cellular.
    8. **Thermal**: Peak thermal_status across samples.
    9. **Launch complete**: Find launch marker, compute startedAt - first_timestamp.

    Use Rust `f64` for all floating point, `i64` for integers. Where Dart uses `int?`, use `Option<i64>`. Where Dart uses `double?`, use `Option<f64>`. Use `Option::None` instead of null. Implement `compute_marker_stats()` and `compute_region_stats()` with same pattern as Dart (they share computation logic).

    FPS analytics port (separate fn `compute_fps_stats()`):
    - Compute sorted Vec<f64> of non-null FPS values
    - Median: middle value (or average of two middle)
    - Min: first sorted element
    - Max: last sorted element
    - 1% low: value at index `(len as f64 * 0.01).floor()`
    - Stability: `(fps_values.iter().filter(|v| **v > 30.0).count() as f64 / len as f64) * 100.0` if len > 0
    - P95 frame time: frame_times sorted, value at 95th percentile; frame_time = 1000.0 / fps (handle divide by zero)
    - Variability index: standard deviation / mean (coefficient of variation)
    - Histogram: JSON string of bucket counts (0-15, 16-30, 31-45, 46-60, 61+)
