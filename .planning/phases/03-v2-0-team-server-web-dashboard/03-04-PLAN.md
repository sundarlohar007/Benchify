    **Step A — Implement server-side trend aggregation queries:**
    Fill in `db/src/trend_queries.rs` with aggregation queries that extract stats from `session_stats` JSONB column:

    ```rust
    pub async fn get_fps_trends(
        pool: &Pool, user_id: Uuid, start_date: Option<NaiveDate>,
        end_date: Option<NaiveDate>, app_name: Option<&str>,
    ) -> Result<Vec<FpsTrendPoint>, AppError>
    ```

    SQL pattern for each trend:
    ```sql
    SELECT
        started_at,
        (session_stats->>'fpsMedian')::FLOAT as fps_median,
        (session_stats->>'fpsStability')::FLOAT as fps_stability,
        (session_stats->>'fpsMin')::FLOAT as fps_min,
        (session_stats->>'fpsMax')::FLOAT as fps_max,
        app_name,
        device_model
    FROM sessions
    WHERE user_id = $1
      AND ($2::DATE IS NULL OR started_at::DATE >= $2)
      AND ($3::DATE IS NULL OR started_at::DATE <= $3)
      AND ($4::VARCHAR IS NULL OR app_name = $4)
    ORDER BY started_at ASC
    ```

    Same pattern for CPU (cpuAvgPct, cpuPeakPct), Memory (memoryAvgKb, memoryPeakKb, memGrowthKb), Battery (batteryDrainPct, mahConsumed, avgPowerMw), Network (netTotalTxKb, netTotalRxKb, netWifiAvgKbps, netCellularAvgKbps).

    Add a `pub async fn get_available_apps(pool: &Pool, user_id: Uuid) -> Result<Vec<String>, AppError>` query for the app filter dropdown.
