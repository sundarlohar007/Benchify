The existing `sessions` table has `project_id` (VARCHAR) and `collection_id` (UUID) columns. The new `team_projects` table uses UUID primary keys. Either: (a) migrate session.project_id to reference team_projects.id via UUID, or (b) keep project_id as a free-text field and add a new `team_project_id` UUID FK column. Decision: use approach (b) — add a new nullable `team_project_id UUID REFERENCES team_projects(id)` column to sessions. Keep existing `project_id` VARCHAR for backward compatibility. This avoids breaking existing session data.

The existing `docker-compose.yml` has two services: `db` (postgres:17-alpine) and `server` (build from Dockerfile). Plan 02 extends this with an `nginx` service, environment-specific overrides, and a `docker-compose.prod.yml` override file.

The audit middleware pattern follows the same approach as auth_middleware: `from_fn_with_state`. However, audit logging should NOT be a middleware interceptor for ALL requests (it would capture GET requests unnecessarily). Instead, audit events are explicitly recorded in route handlers after successful mutating operations. A helper function `record_audit_event()` is called from route handlers.

-- ============================================================
-- Audit Events (V35-06)

CREATE TABLE audit_events (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    event_type VARCHAR(50) NOT NULL,
    event_category VARCHAR(30) NOT NULL CHECK (event_category IN ('auth', 'session', 'user', 'config', 'team', 'export', 'system')),

    target_type VARCHAR(50),

    details JSONB NOT NULL DEFAULT '{}',

CREATE INDEX idx_audit_events_event_category ON audit_events(event_category);
CREATE INDEX idx_audit_events_actor_id ON audit_events(actor_id);
CREATE INDEX idx_audit_events_created_at ON audit_events(created_at DESC);
CREATE INDEX idx_audit_events_target ON audit_events(target_type, target_id);

-- ============================================================
-- Team Organizations (V35-09)
-- ============================================================
CREATE TABLE team_orgs (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    name VARCHAR(255) NOT NULL,
    slug VARCHAR(255) NOT NULL UNIQUE,
    description TEXT,
    is_active BOOLEAN NOT NULL DEFAULT TRUE,
    settings JSONB NOT NULL DEFAULT '{}',
    created_by UUID NOT NULL REFERENCES users(id),
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_team_orgs_slug ON team_orgs(slug);

-- Team Projects (V35-09)
CREATE TABLE team_projects (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    org_id UUID NOT NULL REFERENCES team_orgs(id) ON DELETE CASCADE,
    name VARCHAR(255) NOT NULL,
    slug VARCHAR(255) NOT NULL,
    description TEXT,
    is_active BOOLEAN NOT NULL DEFAULT TRUE,
    created_by UUID NOT NULL REFERENCES users(id),
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE(org_id, slug)
);
CREATE INDEX idx_team_projects_org_id ON team_projects(org_id);

-- Team Membership (V35-09)
CREATE TABLE team_membership (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    org_id UUID NOT NULL REFERENCES team_orgs(id) ON DELETE CASCADE,
