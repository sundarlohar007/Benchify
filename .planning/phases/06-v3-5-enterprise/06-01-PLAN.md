**WARNING — SAML crates limitation**: There is no mature, maintained SAML 2.0 SP crate for the Rust/Axum ecosystem. **OIDC utility** (`server/src/utils/oidc.rs`):
- Use `openidconnect` crate (v4)
- `discover_oidc_provider(issuer_url: &str) -> Result<CoreProviderMetadata>` — performs OIDC discovery at `{issuer_url}/.well-known/openid-configuration`
- `build_oidc_client(metadata, client_id, client_secret, redirect_url) -> CoreClient` — creates OIDC client with HS256/RSA256 token verification
- `build_authorize_url(client, state, nonce) -> (String, String, String)` — returns (authorize_url, csrf_state, pkce_verifier). Uses PKCE with S256 challenge. Scope: openid profile email.
- `exchange_code(client, code, pkce_verifier, nonce) -> CoreIdTokenClaims` — exchanges authorization code for tokens, verifies id_token nonce
- `extract_oidc_attributes(claims) -> (String, Option<String>)` — returns (email, display_name) from claims
- Error handling: map openidconnect errors to AppError variants

**SAML utility** (`server/src/utils/saml.rs`):
- `build_authn_request(idp_sso_url, sp_entity_id, acs_url) -> (String, String)` — returns (redirect_url_with_saml_request, relay_state). Generates XML AuthnRequest: issuer=sp_entity_id, destination=idp_sso_url, AssertionConsumerServiceURL=acs_url, ProtocolBinding=HTTP-POST. Deflate + base64 encode. Append as `?SAMLRequest=...&RelayState=...` to idp_sso_url.
- `validate_saml_response(saml_response_b64, idp_signing_cert_pem) -> Result<SAMLAssertion>` — Base64 decode → XML parse → validate: (a) Response Destination matches ACS URL, (b) Response IssueInstant within clock skew (5 min), (c) Assertion has not expired (NotOnOrAfter), (d) Assertion SubjectConfirmationData Recipient matches ACS URL, (e) Signature validates against IdP certificate using RSA-SHA256, (f) Conditions Audience matches SP entity ID. Return extracted assertion data.
- `extract_saml_attributes(assertion) -> (String, Option<String>)` — parses AttributeStatement for email (Attribute Name="email" or Name="urn:oid:0.9.2342.19200300.100.1.3") and display_name (Name="displayName" or "urn:oid:2.16.840.1.113730.3.1.241"). Falls back to NameID if no email attribute.
- Use `quick-xml` crate (0.37+) for XML. Use ring (already in dep tree via rustls) for signature verification: RSA-PKCS1-SHA256.
- For deflate: use `flate2` crate with `Compression::default()` for encoding, `Decompress::new(true)` for decoding.
- RelayState: generate random 32-byte hex string, validate on callback roundtrip.
- SAMLResponse arrives as base64-encoded POST body form field. The ACS endpoint receives `application/x-www-form-urlencoded` with `SAMLResponse` and `RelayState`.

**LDAP utility** (`server/src/utils/ldap.rs`):
- Use `ldap3` crate (0.11+)
- `ldap_authenticate(server_url, bind_dn_template, search_base, user_filter, username, password) -> Result<(String, Option<String>)>` — (a) Connect with StartTLS if URL scheme is ldaps://, (b) Bind with service account (bind_dn, bind_password from config), (c) Search for user: filter = user_filter.replace("{username}", &escaped_username), attributes = ["mail", "displayName", "cn"], (d) Re-bind as found user DN with provided password, (e) Return (email, display_name) from search attributes
- LDAP attribute mapping: email→mail, display_name→displayName (fallback cn)
- Connection timeout: 10 seconds. Pool connections with ldap3 LdapConn (no connection pooling crate needed — create per-request with timeout)
- Support both ldap:// and ldaps:// URL schemes
- Username escaping per RFC 4515: escape *, (, ), \, NUL characters

**SSO routes** (`server/src/routes/sso.rs`):
- `GET /auth/sso/oidc/login?provider={id}` — Load provider config from DB (or AppConfig fallback), build authorize URL with PKCE, store PKCE verifier + CSRF state in a short-lived signed cookie (`sso_oidc_state`), redirect user to IdP
- `GET /auth/sso/oidc/callback?code={code}&state={state}` — Verify CSRF state matches cookie, exchange code for tokens, verify id_token, extract email/name, call `find_or_create_sso_user` (JIT per V35-03), issue JWT access token cookie + refresh token (same format as local auth POST /auth/login), redirect to `/`
- `POST /auth/sso/saml/login?provider={id}` — Load SAML provider config, build AuthnRequest, store relay_state in signed cookie, return 302 redirect to IdP SSO URL
- `POST /auth/sso/saml/acs?provider={id}` — (IdP POSTs SAMLResponse here). Decode SAMLResponse, validate signature against IdP cert, extract attributes, JIT user, issue JWT cookie, redirect to `/`
- `POST /auth/sso/ldap/login` — JSON body: { provider_id, username, password }. LDAP authenticate, JIT user, issue JWT cookie, return AuthResponse JSON (same format as POST /auth/login). If LDAP auth fails, return 401 Unauthorized.

/// Returns an Axum middleware function that enforces the given minimum role.
/// Must be placed AFTER auth_middleware in the middleware stack.
pub fn require_role(required_role: Role) -> impl Fn(...) -> ... {
    |Extension(auth_user): Extension<AuthUser>, mut request: Request, next: Next| async move {
        let user_role = Role::from_str(&auth_user.role)?;
        if !user_role.satisfies(&required_role) {
            return Err(AppError::Forbidden);
        }
        Ok(next.run(request).await)
    }
}
