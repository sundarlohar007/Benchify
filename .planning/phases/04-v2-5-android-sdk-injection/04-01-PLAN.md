<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| APK file input → Desktop app | User provides arbitrary APK file — must validate it's a valid APK/ZIP before decompiling |
| Keystore credentials → Python subprocess | Passwords cross process boundary as CLI args — clear after use |
| Python subprocess stdout → Desktop parser | JSON from subprocess must be validated before parsing |
| Injected APK → Device | Injected code runs with target app's permissions — must not introduce security regressions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | apk_decompiler.py | mitigate | Validate APK magic bytes (PK\x03\x04) before decompile. Reject non-APK files with clear error. |
| T-04-02 | Information Disclosure | resigner.py / injection_service.dart | mitigate | Keystore passwords passed via stdin (not CLI args visible in process list). Clear password strings from memory after use. Do not log passwords. |
| T-04-03 | Elevation of Privilege | smali_patcher.py | mitigate | Only inject code into Application.onCreate() — never modify AndroidManifest permissions beyond the 5 documented ones. Verify manifest patch is additive-only via diff check. |
| T-04-04 | Spoofing | verifier.py | mitigate | apksigner verify checks signature chain before ADB install. Only proceed past apksigner step if signatures validate. |
| T-04-05 | Denial of Service | injection_service.dart | mitigate | Subprocess timeout at 5 minutes. Kill stuck Python processes via SIGTERM → SIGKILL (same pattern as IosService). |
| T-04-06 | Information Disclosure | injection_screen.dart | mitigate | Keystore file path and password fields: password fields use `obscureText: true`. Keystore path stored in shared_preferences only if user opts in ("Remember keystore" checkbox). |
</threat_model>
