<objective>
Build the PerformanceBench native SDK as a Rust shared library (.so) compiled via cargo-ndk for all Android ABIs. The SDK replaces external ADB metric collection with in-app hooks — Choreographer for FPS, /proc/self/stat for CPU, ActivityManager for memory, GLSurfaceView for GPU, /proc/pid/net/dev for network. SDK streams JSON MetricSample objects newline-delimited over a local TCP server on port 8080. The Android Java shim provides an FPS overlay pill widget (color-coded, draggable, tappable) managed by a foreground service. The desktop app gains an SdkStreamService that connects to the forwarded ADB port and parses the SDK JSON stream into MetricSample objects.

Purpose: Enable in-app profiling that works without ADB shell commands — the SDK runs inside the target app and collects all metrics internally via native hooks and /proc filesystem reads. This gives deeper access than external ADB shell parsing and eliminates the need for device-level shell access.
Output: Compiled Rust .so files for all ABIs, Android overlay widget, TCP streaming service, and desktop stream adapter.
</objective>

    Key functions:
    - `nativeInit` — Spawns tokio runtime (or std::thread) for TCP server. Starts metric collection timer at 1Hz. Returns immediately (non-blocking).
    - `nativeStart` — Begins streaming metrics. If already started, no-op.
    - `nativeStop` — Stops metric collection, closes TCP connections, joins threads.
    - `nativeGetStats` — Returns current metrics snapshot as JSON string.

    **src/transport.rs** — TCP server on 127.0.0.1:8080 (per D-11):
    - Uses `std::net::TcpListener::bind("127.0.0.1:8080")`.
    - Accepts one connection at a time (single desktop client).
    - Writes newline-delimited JSON — one MetricSample per line.
    - JSON field names use snake_case matching `MetricSample.fromMap` exactly: `session_id`, `timestamp`, `fps`, `cpu_app_pct`, `memory_pss_kb`, `net_tx_bytes`, `net_rx_bytes`, `gpu_pct`, etc. — all 48 fields.
    - Non-blocking writes using `set_nonblocking(true)`. If client disconnects, queue samples and reconnect when client reconnects.
    - Always-on from app start (per D-13): TCP server starts in `nativeInit`, begins emitting samples immediately. Desktop connects anytime.

    **src/metrics/fps.rs** — Choreographer frame callback hook via JNI:
    - Register `Choreographer.FrameCallback` via JNI `postFrameCallback`.
    - Compute FPS as 1e9 / avg_frame_delta_ns over a 1-second window.
    - Classify jank: frame delta > 2x vsync period = small jank, > 4x = big jank.
    - Output fields: `fps`, `jank_count`, `jank_small_count`, `jank_big_count`, `jank_ratio_count`, `frametimes_json` (array of last 60 frame deltas in ms).

    **src/metrics/cpu.rs** — CPU via `/proc/self/stat`:
    - Read utime + stime from fields 14-15 (1-indexed) of /proc/self/stat.
    - Compute delta per second, divide by clock ticks (_SC_CLK_TCK via libc::sysconf), normalize to percentage.
    - Read `/proc/stat` for system-wide CPU time to compute cpu_system_pct.
    - Read `/sys/devices/system/cpu/cpuN/cpufreq/scaling_cur_freq` for per-core frequencies.
    - Output fields: `cpu_app_pct`, `cpu_system_pct`, `cpu_app_pct_freq_norm`, `cpu_cores` (count), `cpu_core_states_json`, `cpu_core_freqs_json`.

    **src/metrics/memory.rs** — Memory via ActivityManager JNI:
    - Call `ActivityManager.getProcessMemoryInfo([pid])` through JNI.
    - Extract `totalPss`, `dalvikPss`, `nativePss`, `otherPss`.
    - Maps to: `memory_pss_kb` (totalPss), `memory_java_kb` (dalvikPss), `memory_native_kb` (nativePss), `memory_system_kb` (otherPss).
    - Fallback: read `/proc/self/status` VmRSS if ActivityManager unavailable.

    **src/metrics/network.rs** — Per-process network via `/proc/self/net/dev` (per D-16):
    - Read `/proc/self/net/dev`, parse interface lines.
    - Track cumulative TX/RX bytes per interface (wlan0, rmnet_data0, etc.).
    - Compute per-second deltas for each interface.
    - Classify interfaces: wifi (starts with "wlan"), cellular (starts with "rmnet"), other.
    - Output fields: `net_tx_bytes`, `net_rx_bytes`, `net_wifi_tx_bytes`, `net_wifi_rx_bytes`, `net_cellular_tx_bytes`, `net_cellular_rx_bytes`, `net_other_tx_bytes`, `net_other_rx_bytes`.

    **src/metrics/gpu.rs** — GPU utilization via sysfs (per D-10):
    - Read `/sys/class/kgsl/kgsl-3d0/gpubusy` (Adreno) or `/sys/class/misc/mali0/device/utilization` (Mali).
    - Parse GPU busy percentage.
    - Output fields: `gpu_pct`.
