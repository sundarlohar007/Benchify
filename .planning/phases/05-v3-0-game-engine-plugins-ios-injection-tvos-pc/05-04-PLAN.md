    4. `pcprobe/src/ipc.rs` — TCP + named pipe IPC server:
       - Two transport modes (per §19.1):
         - **TCP**: `TcpListener::bind("{host}:{port}")` — default `127.0.0.1:27184` for same-machine, configurable for LAN
         - **Named pipe**: `\\.\pipe\pb-pcprobe` — Windows-only, same-machine only, used when TCP port conflicts
       - `pub struct IpcServer { listener: TcpListener, clients: Vec<TcpStream> }`
       - `pub fn start_server(host: &str, port: u16) -> Result<IpcServer, anyhow::Error>`:
         - Binds TCP listener
         - Spawns tokio task: accept connections, handle each client in separate task
       - `pub async fn broadcast_sample(server: &IpcServer, sample: &MetricSample) -> Result<()>`:
         - JSON-serialize MetricSample (uses serde, same format as Phase 4 transport.rs)
         - Write newline-delimited JSON to all connected clients
         - Non-blocking: if client disconnected, remove from list
       - `pub async fn broadcast_event(server: &IpcServer, event_json: &str) -> Result<()>`:
         - Same pattern for marker events, status messages
       - Client protocol:
         - Each line = one JSON object (same NDJSON pattern as Android SDK port 8080)
         - Commands FROM host TO probe (first field: `"cmd"`):
           - `{"cmd":"START","session_id":"..."}` → starts collection
           - `{"cmd":"STOP"}` → stops collection
           - `{"cmd":"PAUSE"}` → pauses collection (keeps IPC alive)
           - `{"cmd":"RESUME"}` → resumes collection
           - `{"cmd":"MARKER","name":"boss_fight","note":""}` → creates marker
           - `{"cmd":"SCREENSHOT"}` → triggers screenshot (PC: full desktop capture)
           - `{"cmd":"VIDEO_START"}` → starts video recording (if enabled)
           - `{"cmd":"VIDEO_STOP"}` → stops video recording
           - `{"cmd":"STATUS"}` → returns probe status JSON
         - Responses FROM probe TO host:
           - MetricSample JSON (same structure as Phase 4 models.rs)
           - Marker event JSON: `{"type":"marker","name":"...","ts":...}`
           - Status JSON: `{"status":"running","process":"game.exe","pid":1234,"uptime_s":120,...}`

    5. `pcprobe/src/discovery.rs` — mDNS auto-discovery (per §19.7):
       - `pub fn advertise_pcprobe(port: u16) -> Result<mdns_sd::ServiceDaemon, anyhow::Error>`:
         - Registers `_pb-pcprobe._tcp.local.` service via `mdns-sd` crate
         - Properties: `port={port}`, `version=3.0.0`, `hostname={hostname}`
         - Returns daemon handle for cleanup
       - `pub fn discover_hosts() -> Result<Vec<DiscoveredHost>, anyhow::Error>`:
         - Browsers for `_pb-pcprobe._tcp.local.` services
         - Returns list of `DiscoveredHost { address, port, hostname, version }`
         - Used by desktop to auto-discover probes on LAN
       - Falls back to manual `--host` if mDNS unavailable (Windows without Bonjour, strict firewalls)

    6. `pcprobe/src/collector.rs` — 1Hz collection glue (wires Plan 05-03 modules into binary):
       - `pub fn run_collector(args: Args, ipc: Arc<IpcServer>) -> Result<JoinHandle<()>>`:
         - Resolve process: if `--process-id` given, use it; if `--process-name`, find PID via `sysinfo` crate
         - Create `pc_metrics::collector::PcCollector::new(process_name, pid, dxgi_method, etw)`
         - Spawn thread: loop at 1Hz (sleep for 1s - elapsed collection time):
           - `let sample = collector.tick()?;`
           - `ipc.broadcast_sample(&sample).await?;`
           - If video enabled: feed frame timestamps to video sync
         - Returns `JoinHandle` for async cancellation
       - Error handling: if process exits, send `{"type":"error","code":"PROCESS_EXITED"}` to host, keep probe alive for reattach

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Desktop host ↔ pb-pcprobe | TCP 127.0.0.1:27184 or named pipe — same-machine trust |
| pb-pcprobe ↔ Game process | DXGI hook injection (read-only frame timestamps) |
| pb-pcprobe ↔ Windows PDH API | User-mode counter queries |
| pb-pcprobe ↔ ETW kernel | Requires admin — explicit user opt-in |
| Desktop ↔ LAN probe | TCP over LAN — user-configured, opt-in |
| Video capture ↔ Display/GPU | Native capture APIs with system permission prompts |
| ffmpeg subprocess ↔ Host filesystem | Writes video chunks to user-specified directory |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-16 | Spoofing | mDNS discovery | mitigate | Probe advertises with unique hostname + version. Desktop verifies probe version compatibility before connecting. |
| T-05-17 | Tampering | IPC commands from host to probe | accept | Local-only IPC (127.0.0.1 or named pipe). No authentication needed — same-machine trust boundary. LAN mode is opt-in with warning. |
| T-05-18 | Information Disclosure | Video capture | mitigate | System permission prompts on Windows (GraphicsCapturePicker) and macOS (TCC). No audio capture. Video stored locally, not auto-uploaded. |
| T-05-19 | Information Disclosure | LAN probe streaming | mitigate | LAN mode disabled by default (127.0.0.1 only). When enabled, user warned: "Exposing performance data to LAN." |
| T-05-20 | Elevation of Privilege | pb-pcprobe self-elevation for ETW | mitigate | Probe runs as standard user by default. ETW elevation only when user explicitly enables `--etw` flag. Probe checks admin status and prints clear message if insufficient. Never auto-elevates. |
| T-05-21 | Denial of Service | ffmpeg subprocess resource exhaustion | mitigate | ffmpeg CPU priority set to BELOW_NORMAL. Chunk duration caps at 5 minutes. Disk-full guard: recording auto-stops when < 100 MB free (per §32.6). |
