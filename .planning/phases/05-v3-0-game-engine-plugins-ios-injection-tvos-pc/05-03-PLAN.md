       API functions:
       - `pub fn open_query(process_name: &str, include_gpu: bool) -> Result<PdhQuery, String>`:
         - Calls `PdhOpenQueryW` to create query handle
         - For each metric in §19.2 table, calls `PdhAddEnglishCounterW` with counter path:
           - CPU: `\Process({process_name})\% Processor Time`
           - CPU per-core: `\Processor(0..N)\% Processor Time` (N = `num_cpus::get()`)
           - Working set: `\Process({process_name})\Working Set`
           - Private bytes: `\Process({process_name})\Private Bytes`
           - Page faults: `\Process({process_name})\Page Faults/sec`
           - Disk read: `\Process({process_name})\IO Read Bytes/sec`
           - Disk write: `\Process({process_name})\IO Write Bytes/sec`
           - Thread count: `\Process({process_name})\Thread Count`
           - Handle count: `\Process({process_name})\Handle Count`
           - Network: `\Network Interface(*)\Bytes Received/sec` + `\Bytes Sent/sec`
           - GPU usage: `\GPU Engine(*engtype_3D)\Utilization Percentage` (if gpu enabled)
           - GPU dedicated: `\GPU Process Memory(*)\Dedicated Usage`
           - GPU shared: `\GPU Process Memory(*)\Shared Usage`
         - All fallible: if counter doesn't exist (e.g., no GPU), set to None, log warning, continue
       
       - `pub fn collect_sample(query: &PdhQuery) -> Result<PcMetricsSnapshot, String>`:
         - Calls `PdhCollectQueryData` to refresh all counters
         - For each counter, calls `PdhGetFormattedCounterValue(PDH_FMT_DOUBLE | PDH_FMT_LARGE)` to read value
         - Maps each counter name to the correct `PcMetricsSnapshot` field
         - CPU % is raw `\Process()\% Processor Time` — ALREADY per-process, does NOT divide by core count (PDH `% Processor Time` for Process object = total CPU time across all cores / elapsed time / 100ns = fraction. Must then multiply by 100 / num_cores for a percentage relative to single core, OR leave raw for multi-core display per §19.2 note "÷ logical core count")
         - Disk/network: cumulative bytes → stored as cumulative in snapshot (rate calculation done by consumer at next sample using (current - previous) / interval_s)
         - Returns snapshot with `timestamp: SystemTime::now()`
       
       - `pub fn close_query(query: PdhQuery)`:
         - Calls `PdhCloseQuery` on each counter, then the query handle
       
       - Platform gating: All PDH code is `#[cfg(windows)]`. Non-Windows targets get stub module that returns `Err("PDH is Windows-only")`.

    3. `dxgi.rs` — DXGI Present hook for game FPS (per §19.3):

       Two methods implemented as Rust library code:

       **Method A — Detours injection** (preferred, low overhead):
       - `pub fn build_dxgi_hook_dll() -> Result<Vec<u8>, String>`:
         - Returns pre-compiled `pb-pcprobe-dx.dll` bytes (the DLL that gets injected)
         - This DLL hook logic (conceptually in a separate crate or embedded bytes):
           - Hooks `IDXGISwapChain::Present` via Microsoft Detours
           - On each Present call: `QueryPerformanceCounter()` → write timestamp to shared memory ring buffer
           - Ring buffer: 64KB shared memory, lock-free single-producer (game) / single-consumer (probe)
         - The Rust code for the hook DLL is minimal — a Trampoline function saves QPC, calls original Present
       
       - `pub fn inject_dx_hook(process_id: u32) -> Result<SharedMemoryHandle, String>`:
         - Calls `OpenProcess` + `VirtualAllocEx` + `WriteProcessMemory` (path string) + `CreateRemoteThread` (LoadLibraryW)
         - All via `windows-rs` `Win32_System_Threading` and `Win32_System_Memory`
         - Returns handle to shared memory ring buffer
       
       - `pub fn read_frame_deltas(ring_buffer: &SharedMemory) -> Vec<u64>`:
         - Reads ring buffer from head to tail, computes QPC deltas between consecutive entries
         - Converts QPC deltas to nanoseconds using `QueryPerformanceFrequency`
         - Returns `Vec<u64>` of inter-frame intervals in nanoseconds

       **Method B — PresentMon integration** (fallback, no injection):
       - `pub fn start_presentmon(process_name: &str) -> Result<PresentMonSession, String>`:
         - Ships `presentmon.exe` (Microsoft MIT-licensed) in `tools/presentmon/`
         - Spawns: `presentmon.exe --process_name {name} --output_stdout --timed 1`
         - PresentMon outputs CSV to stdout with frame timestamps
         - Parse CSV lines: `FrameTime,TimeInSeconds,MsBetweenPresents,...` → extract `MsBetweenPresents`
         - Convert ms deltas to ns for `compute_fps` input
       
       - `pub fn measure_fps_presentmon(session: &PresentMonSession) -> Result<FpsResult, String>`:
         - Reads accumulated frame deltas from PresentMon session
         - Calls `crate::metrics::fps::analyze_fps(frame_deltas_ns)` (reuses Phase 4 FPS computation exactly)
         - Returns `FpsResult` with fps, jank counts, frametimes_json
       
       - `pub fn stop_presentmon(session: PresentMonSession)`:
         - Terminates presentmon.exe subprocess
