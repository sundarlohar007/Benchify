    **Part A — Python iOS IPA injection engine** (per D-06, D-07):

    1. `ipa_injector.py` — Main injection orchestrator:
       - `def inject_dylib(ipa_path: str, output_path: str, signing_method: str, apple_id: Optional[str], team_id: Optional[str], cert_path: Optional[str]) -> InjectionResult`
       - **Step 1 — Unzip IPA**: Extract to temp directory. IPA is just a zip with `Payload/AppName.app/`.
       - **Step 2 — FairPlay check**: Read `Payload/AppName.app/AppName` Mach-O header. Check `LC_ENCRYPTION_INFO.cryptid`. If `cryptid != 0` → abort with clear error: "This IPA is encrypted (FairPlay DRM). Studio-provided unencrypted IPAs only. App Store IPAs cannot be injected."
       - **Step 3 — dylib placement**: Copy `PerformanceBench.framework` into `Payload/AppName.app/Frameworks/PerformanceBench.framework/`. The framework contains: `PerformanceBench` (dylib binary), `Info.plist`, `Modules/`. Pre-built from Rust core compiled as `cdylib` targeting `aarch64-apple-ios`.
       - **Step 4 — Info.plist patching**: Read `Payload/AppName.app/Info.plist`. Insert into `CFBundleExecutable` or `UILaunchStoryboardName` adjacent: Add `MinimumOSVersion: 14.0` if lower. Ensure `UIDeviceFamily` includes 1 (iPhone). NO network usage descriptions added (privacy contract — no network access from SDK).
       - **Step 5 — Load command insertion**: Use `macholib` (Python) or `install_name_tool` to add `@rpath/PerformanceBench.framework/PerformanceBench` as a load command in the main executable. Alternatively: insert `LC_LOAD_DYLIB` command pointing to `@executable_path/Frameworks/PerformanceBench.framework/PerformanceBench`.
       - **Step 6 — Re-sign**: Call `apple_signing.py` with chosen method.
       - **Step 7 — Verify**: Call `ipa_verifier.py`.
       - **Step 8 — Repack**: Zip `Payload/` into new `.ipa` file at `output_path`.
       - Return `InjectionResult(success, output_path, signing_method_used, warnings, steps_completed)`.
       - CLI integration: Add `ipa-inject` subcommand to `injector_cli.py`:
         - `python injector_cli.py ipa-inject --input app.ipa --output app-injected.ipa --signing free --apple-id user@icloud.com`
         - `--signing` options: `free`, `paid`, `cert`

    2. `apple_signing.py` — Signing method auto-detection and execution (per D-06):
       - `def detect_available_methods() -> List[SigningMethod]`:
         - Check 1: `xcrun altool --list-providers` — if succeeds, Apple ID auth works (free or paid)
         - Check 2: Look for `~/Library/MobileDevice/Provisioning Profiles/*.mobileprovision` — paid developer account exists
         - Check 3: `security find-identity -v -p codesigning` — user-installed signing certificates
         - Returns list of available `SigningMethod` enums: `FREE_APPLE_ID`, `PAID_DEVELOPER`, `USER_CERTIFICATE`
       - `def free_apple_id_sign(ipa_path: str, apple_id: str, app_specific_password: str) -> SignResult`:
         - Uses `xcrun altool --store-password-in-keychain-item` to store app-specific password
         - Uses `xcrun codesign --force --deep --sign "Apple Development: {apple_id}" --entitlements entitlements.plist Payload/AppName.app`
         - Creates minimal `entitlements.plist` with `get-task-allow: true` (required for dev signing)
         - 7-day expiry documented in UI warning and returned in `SignResult.warnings`
       - `def paid_developer_sign(ipa_path: str, team_id: str, profile_path: str) -> SignResult`:
         - Uses `xcrun codesign --sign "{team_id}" --embed "{profile_path}"` pattern
       - `def user_certificate_sign(ipa_path: str, cert_identity: str) -> SignResult`:
         - Uses `security find-identity` matched identity hash
       - Credential security: Apple ID password stored in macOS Keychain via `security add-generic-password`, never in plaintext config files. App-specific password required (not main Apple ID password).
       - CLI: `xcrun altool` integration for notarization check (free Apple ID: no notarization, sideload only).

    3. `ipa_verifier.py` — Post-injection verification:
       - `def verify_injection(ipa_path: str) -> VerificationResult`:

         - Check 3: `otool -L Payload/AppName.app/AppName` shows PerformanceBench in load commands
         - Check 4: `codesign -dv Payload/AppName.app` returns valid signature
         - Check 5: `codesign --verify --deep --strict Payload/AppName.app` passes
         - Returns `VerificationResult(all_passed, checks: List[CheckResult])`

       - **Device discovery**: Uses pyidevice `usbmux` device list, filtering for `DeviceClass: AppleTV`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Desktop ↔ Python injector | Subprocess boundary — Python script spawned from Flutter |
| Desktop ↔ pyidevice (tvOS collector) | Local USB/WiFi connection to Apple TV |
| IPA injection ↔ Apple ID | Keychain credential storage boundary |
| Injected IPA ↔ iOS device | Sideload boundary — app runs on user device |
| tvOS collector ↔ Apple TV | pyidevice DTXProtocol over USB-C |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-06 | Spoofing | apple_signing.py altool calls | mitigate | Only use `xcrun altool` (Apple-signed binary). Validate altool path is from Xcode installation (not PATH-injected). |
| T-05-07 | Information Disclosure | Apple ID credentials | mitigate | Credentials stored in macOS Keychain via `security add-generic-password`. App-specific password enforced (not main Apple ID password). Never stored in config files or shared_preferences. |
| T-05-08 | Tampering | Injected IPA dylib | accept | Injected dylib modifies app binary. User accepts this risk — documented as "sideload modification" in UI. Dylib is localhost-only (no network capability). |
| T-05-09 | Elevation of Privilege | pyidevice tvOS connection | accept | pyidevice runs with user privileges only. tvOS DTXProtocol requires device trust (user must accept "Trust this computer?" on Apple TV). |
| T-05-10 | Denial of Service | Encrypted IPA injection attempt | mitigate | Early FairPlay detection (cryptid check before any modification). Abort immediately with clear error — no wasted processing. |
</threat_model>
